Los párrafos contienen texto. No debería trabarse ningún texto sin estar anidado (dentro de una etiqueta) en un elemento HTML. Es muy importante considerar el valor semántico de los elementos. En HTML, el término "semántico" se refiere al uso de etiquetas que no solo describen la apariencia de un elemento, sino que también comunican el propósito o el significado tanto para los navegadores, como para los motores de búsqueda y otros sistemas de asistencia como los lectores de pantalla. Las etiquetas nos ayudan a que el contenido sea más comprensible y estructurado, lo que mejora la accesibilidad y la optimización en los motores de búsqueda.

Los elementos se clasifican en dos tipos principales: elementos de bloque y elementos de línea.
Elementos de bloque: siempre comienzan en una nueva línea.
Pueden contener otros elementos de bloque o elementos en línea.
Ejemplos: etiquetas h (h1...h6), p, div, form, listas

Elementos de línea: no comienzan en una nueva línea y solo ocupan el ancho necesario.
Pueden contener texto u otros elementos en línea. Ejemplos: span, a, img, strong, em, label, input

Algunos ejemplos básicos:
<'b'> (bold) -> negrita
<'i'> (italic) -> cursiva
<'u'> (underline) -> subrayado
Estos elementos ya no se utilizan, el estilo se define desde css
Y la importancia semántica, la vamos a manejar con los elementos: strong -> texto resaltado en negrita (considera un peso especial de importancia)
em -> texto en italic (considera un énfasis especial de importancia)
